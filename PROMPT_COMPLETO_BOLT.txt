─────────────────────────────────────────
TABLA: products
─────────────────────────────────────────

13. DETALLES IMPORTANTES DE IMPLEMENTACIÓN
═══════════════════════════════════════════

MANEJO DE ESTADO:
- Usar useState y useEffect de React
- No usar librerías de gestión de estado global
- Pasar props necesarias entre componentes

LLAMADAS A SUPABASE:
- Usar supabase.from('tabla').select/insert/update/delete
- Siempre usar .maybeSingle() para queries que esperan 0 o 1 resultado
- Manejar errores con try/catch
- Mostrar mensajes de error al usuario

VALIDACIONES:
- Validar campos requeridos
- Validar tipos de datos
- Validar unicidad donde corresponda
- Mostrar errores claros al usuario

FECHAS Y HORAS:
- Usar zona horaria de Argentina (America/Argentina/Buenos_Aires)
- Formato de visualización: DD/MM/YYYY HH:mm:ss
- Almacenar en DB como timestamptz

FORMATO DE NÚMEROS:
- Precios y montos: 2 decimales
- Usar .toFixed(2) para mostrar
- Símbolo $ antes del número

RESPONSIVE DESIGN:
- Mobile first
- Usar grid de Tailwind

14. FLUJOS IMPORTANTES
═══════════════════════════════════════════

FLUJO COMPLETO DE UNA VENTA:
1. Usuario inicia turno con efectivo inicial
2. Entra al módulo Ventas
3. Busca y agrega productos al carrito
4. Ajusta cantidades
5. Selecciona método de pago (o pago múltiple)
6. Opcionalmente ingresa datos de cliente
7. Confirma venta
8. Sistema:
   - Genera número de venta único
   - Crea registro en sales con items completos
   - Actualiza stock de cada producto
   - Crea transacción en cash_transactions tipo income
   - Limpia carrito
9. Usuario puede continuar vendiendo

FLUJO DE CIERRE DE TURNO:
1. Usuario va a módulo Caja
2. Click en "Cerrar Turno"
3. Sistema muestra:
   - Efectivo inicial
   - Ventas en efectivo del turno
   - Egresos en efectivo del turno
   - Efectivo esperado = inicial + ventas - egresos
4. Usuario ingresa efectivo real contado
5. Sistema calcula diferencia
6. Usuario confirma cierre
7. Sistema:
   - Actualiza turno con end_date, closing_cash, totales
   - Marca turno como inactivo
   - Vuelve a pantalla de login
8. Siguiente usuario puede iniciar nuevo turno

FLUJO DE BACKUP Y RESTAURACIÓN:
1. Usuario va a Configuración → Backup
2. Click en "Descargar Backup"
3. Sistema descarga JSON con todos los datos
4. Usuario guarda archivo en lugar seguro
5. Si necesita restaurar:
   - Selecciona archivo de backup
   - Click en "Restaurar Backup"
   - Lee advertencias
   - Confirma
   - Sistema borra todo y carga datos del backup
   - Página se recarga
   - Sistema queda como estaba en la fecha del backup
